# 视频无法加载问题排查与解决

## 问题描述

视频博文页面无法加载视频，直接显示"视频加载失败"。

- 网站：`https://91tutu.cc/videos/videotest/videotest1/`
- 对比：同样使用R2存储和HLS技术的其他网站（如 `https://91shenshi.com`）视频可以正常播放

---

## 问题分析过程

### 第一阶段：初步诊断

#### 1.1 检查浏览器控制台错误

打开浏览器开发者工具，发现以下错误：

```
[error] HLS error: hlsError {
  type: networkError, 
  details: manifestLoadError, 
  fatal: true, 
  url: https://video.91tutu.cc/video/videotest/videotest1/video_2025-09-03_00-01-03/playlist.m3u8
}

[error] net::ERR_FAILED https://video.91tutu.cc/video/.../playlist.m3u8
```

**错误类型**：`ERR_FAILED` 表示请求完全失败

#### 1.2 对比分析

| 项目 | 普通博文图片 | 视频博文视频 |
|------|-------------|-------------|
| URL | `https://image.91tutu.cc/image/...` | `https://video.91tutu.cc/video/...` |
| 加载状态 | ✅ 正常 | ❌ 失败 |
| 请求方式 | `<img>` 标签直接加载 | HLS.js 通过 XHR 请求 |

**关键发现**：HLS视频使用 `XMLHttpRequest` 请求，受CORS严格限制；而图片通过 `<img>` 标签加载，不受CORS限制。

---

### 第二阶段：CORS配置排查

#### 2.1 检查R2 CORS策略

发现R2 bucket的CORS策略为**空**。

#### 2.2 添加CORS策略

在Cloudflare R2控制台添加以下CORS策略：

```json
[
  {
    "AllowedOrigins": ["https://91tutu.cc"],
    "AllowedMethods": ["GET", "HEAD"],
    "AllowedHeaders": ["*"],
    "MaxAgeSeconds": 3600
  }
]
```

#### 2.3 CORS配置后的变化

- **之前**：直接显示"视频加载失败"
- **之后**：出现视频框，显示旋转加载图标，然后显示"视频加载失败"

**结论**：CORS配置有部分效果，但问题仍未完全解决。

---

### 第三阶段：MIME类型排查（根本原因）

#### 3.1 检查R2文件属性

查看R2中视频文件的属性，发现MIME类型配置错误：

| 文件 | 当前MIME类型 | 正确MIME类型 |
|------|-------------|-------------|
| `playlist.m3u8` | `application/octet-stream` | `application/vnd.apple.mpegurl` |
| `segment_000.ts` | `text/plain; charset=utf-8` | `video/mp2t` |

#### 3.2 MIME类型错误的影响

1. **`.m3u8` 被识别为二进制流**：浏览器无法识别为HLS播放列表
2. **`.ts` 被识别为文本**：浏览器尝试按文本解析视频数据，导致失败

**这是视频无法加载的根本原因！**

---

### 第四阶段：修复上传脚本

#### 4.1 尝试方法1：使用 `--header` 参数

```python
cmd = [
    'rclone', 'copy', local_file, f'{REMOTE_NAME}:{remote_path}',
    '--header', f'Content-Type: {mime_type}'
]
```

**结果**：失败

**错误信息**：
```
SignatureDoesNotMatch: The request signature we calculated does not match 
the signature you provided. Check your secret access key and signing method.
```

**原因**：使用 `--header` 参数时，rclone需要重新计算签名，但R2配置不支持。

#### 4.2 尝试方法2：使用 `--metadata` 参数

```python
cmd = [
    'rclone', 'copy', local_file, f'{REMOTE_NAME}:{remote_path}',
    '--metadata', f'Content-Type={mime_type}'
]
```

**结果**：失败

**错误信息**：
```
Usage: rclone copy source:path dest:path [flags]
```

**原因**：rclone 版本不支持 `--metadata` 参数。

#### 4.3 尝试方法3：使用 `--header-upload` 参数（成功）

```python
cmd = [
    'rclone', 'copy', local_file, f'{REMOTE_NAME}:{remote_dir}',
    '--header-upload', f'Content-Type: {mime_type}'
]
```

**结果**：成功！

---

### 第五阶段：修复其他问题

#### 5.1 Windows路径分隔符问题

**问题**：Windows使用 `\` 作为路径分隔符，但R2需要 `/`

**解决**：
```python
rel_path_unix = rel_path.replace('\\', '/')
```

#### 5.2 文件路径变成目录问题

**问题**：上传后文件路径变成 `playlist.m3u8/playlist.m3u8`

**原因**：`rclone copy` 的目标参数应该是目录，而不是包含文件名的完整路径

**解决**：
```python
# 提取目录部分
remote_dir = os.path.dirname(remote_path)
cmd = ['rclone', 'copy', local_file, f'{REMOTE_NAME}:{remote_dir}']
```

## 最终解决方案

### 修改后的上传脚本关键代码

```python
def get_mime_type(file_path):
    """根据文件扩展名获取MIME类型"""
    ext = os.path.splitext(file_path)[1].lower()
    mime_types = {
        '.m3u8': 'application/vnd.apple.mpegurl',
        '.ts': 'video/mp2t',
        '.json': 'application/json',
    }
    return mime_types.get(ext, None)

def upload_file_with_mime(local_file, remote_path):
    """上传单个文件并设置正确的MIME类型"""
    mime_type = get_mime_type(local_file)
    
    # 提取目录部分
    remote_dir = os.path.dirname(remote_path)
    if not remote_dir:
        remote_dir = "."
    
    # 构建命令
    cmd = ['rclone', 'copy', local_file, f'{REMOTE_NAME}:{remote_dir}']
    
    # 添加MIME类型设置
    if mime_type:
        cmd.extend(['--header-upload', f'Content-Type: {mime_type}'])
    
    # 执行上传
    result = subprocess.run(cmd, ...)
    return result.returncode == 0
```

### 验证结果

上传后验证MIME类型：

```bash
$ rclone lsjson small:small/video/.../playlist.m3u8
{"MimeType":"application/vnd.apple.mpegurl"}  # ✅ 正确

$ rclone lsjson small:small/video/.../segment_000.ts
{"MimeType":"video/mp2t"}  # ✅ 正确
```

## 问题总结

### 根本原因

1. **MIME类型配置错误**：R2中HLS文件的Content-Type不正确
2. **CORS策略缺失**：跨域请求被阻止

### 解决步骤

1. ✅ 配置R2 CORS策略
2. ✅ 修改上传脚本，自动设置正确的MIME类型
3. ✅ 使用 `--header-upload` 参数设置Content-Type
4. ✅ 修复Windows路径分隔符问题
5. ✅ 修复rclone目标路径问题
6. ✅ 重新上传所有视频文件

### 经验教训

1. **HLS视频对MIME类型要求严格**：必须正确设置 `.m3u8` 和 `.ts` 文件的Content-Type
2. **CORS是HLS跨域播放的必要条件**：但不是唯一条件
3. **rclone参数选择很重要**：不同参数对签名计算有不同影响
4. **对比分析是排查问题的有效方法**：通过对比正常和异常案例快速定位问题
