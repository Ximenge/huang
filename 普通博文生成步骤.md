## 关键文件说明

### conversion_record.json

**位置**: `c:\huang\Image\conversion_record.json`

**作用**: 记录已处理的文件夹，避免重复转换

**格式**:
```json
{
  "C:\\huang\\image\\yuuhui玉汇 - 痴女日记 [101P1.88GB]": {
    "converted_count": 104,
    "target_directory": "C:\\huang\\image-1\\yuuhui玉汇 - 痴女日记 [101P1.88GB]",
    "timestamp": 1772227040.7609172
  }
}
```

**字段说明**:
- `converted_count`: 转换的文件数量
- `target_directory`: 目标目录路径
- `timestamp`: 转换时间戳

## 注意事项

### 1. 转换记录管理
- `conversion_record.json` 只在步骤1（转换）时使用
- 步骤2（上传）和步骤3（生成博文）不检查此记录
- 这样可以确保即使文件夹已转换，如果未上传或未生成博文，仍然可以执行相应步骤

### 2. 文件命名规范
- 转换后的WebP文件按顺序重命名：001.webp, 002.webp, ...
- Markdown文件使用拼音slug命名
- R2远程目录使用原始文件夹名

### 3. 错误处理
- 如果某个步骤失败，检查错误信息
- 转换失败：检查原始图片格式和大小
- 上传失败：检查rclone配置和网络连接
- 生成失败：检查模板配置和路径
- 构建失败：检查依赖和配置文件
- 部署失败：检查wrangler配置和认证

## 常见问题

### Q1: 为什么转换脚本跳过了某些文件夹？
A: 这些文件夹已经在 `conversion_record.json` 中记录，表示已经处理过。

### Q2: 如何重新处理已记录的文件夹？
A: 编辑 `conversion_record.json`，删除对应的记录，然后重新运行转换脚本。

### Q3: 上传失败怎么办？
A: 检查 rclone 配置，确认远程存储连接正常，然后重新运行上传脚本。

### Q4: 如何查看已生成的博文？
A: 博文文件位于 `c:\huang\astro-melody-starter\src\content\posts\` 目录。

### Q5: 部署后如何访问网站？
A: 部署完成后会提供预览URL，也可以访问配置的自定义域名。
