# 发布博文详细步骤

---

## 一、普通博文（图片博文）

### 目录结构说明

```
C:\huang\
├── image/                          # 原始图片目录
│   └── 博文标题/                   # 以博文标题命名的文件夹
│       ├── 图片1.jpg
│       ├── 图片2.png
│       └── ...
│
├── Image-1/                        # 处理后的图片目录（WebP格式）
│   └── 博文标题/
│       ├── 图片1.webp
│       ├── 图片2.webp
│       └── ...
│
└── astro-melody-starter/
    └── src/
        └── content/
            └── posts/              # 博文Markdown文件
                └── 博文标题.md
```

### 步骤详解

#### 步骤1：准备图片

将图片放入 `image/` 目录，以博文标题创建文件夹：

```
image/
└── 眼镜妹/
    ├── photo_001.jpg
    ├── photo_002.png
    └── ...
```

#### 步骤2：转换为 WebP 格式

```powershell
cd C:\huang\image
python convert_to_webp_size_optimized.py
```

#### 步骤3：添加元数据文件

```powershell
cd C:\huang\image
python add_metadata_tags.py
```

脚本功能：
- 遍历 `Image/` 目录下的所有子文件夹
- 检查每个文件夹是否已存在 `post_metadata.json` 文件
- 如果不存在，则创建该文件，内容为：
  ```json
  {
    "category": "R18",
    "tags": [
      "从文件夹名称提取的标签（第一个空格前的部分）"
    ]
  }
  ```
- 如果文件已存在，则跳过该文件夹

#### 步骤4：上传到 R2 存储

```powershell
cd C:\huang\Image-1
python upload_to_r2.py
```

脚本功能：
- 遍历 `image/` 目录下的所有子文件夹
- 将图片转换为 WebP 格式
  - 目标文件大小：200KB以下
  - 最大分辨率：1440px（保持原始比例）
  - 自动调整质量和分辨率以优化文件大小）
- 输出到 `Image-1/` 目录
- 记录转换历史到 `conversion_record.json`

#### 步骤5：上传到 R2 存储

```powershell
cd C:\huang\Image-1
python upload_to_r2.py
```

脚本功能：
- 遍历 `Image-1/` 目录下的所有文件夹
- 使用 rclone 上传到 Cloudflare R2 存储
- 远程路径：`small/博文标题/`
- R2 公开访问地址：`https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/博文标题/`

#### 步骤6：生成博文 Markdown 文件

```powershell
cd C:\huang\Image-1
python generate_markdown.py
```

脚本功能：
- 遍历 `Image-1/` 目录下的所有文件夹
- 为每个文件夹生成对应的 `.md` 文件
- 输出到 `astro-melody-starter/src/content/posts/`
- 自动跳过已存在的博文

生成的 Markdown 文件格式：
```markdown
---
author: Image Gallery
category:
- Gallery
cover: https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/博文标题/第一张图片.webp
coverAlt: 博文标题 gallery image
description: A collection of images from 博文标题
pubDate: 2026-02-28 12:00:00
slug: 博文标题
tags:
- images
- gallery
title: 博文标题
---

# 博文标题

This gallery contains XX images.

## Image Gallery

![图片1.webp](https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/博文标题/图片1.webp)

![图片2.webp](https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/博文标题/图片2.webp)
```

#### 步骤7：验证

打开浏览器访问 http://localhost:4321/ 查看新博文。

## 二、视频博文

### 目录结构说明

```
C:\huang\
├── video/                          # 原始视频目录
│   └── 视频分组/                   # 一级目录：视频分组/系列
│       └── 博文标题/               # 二级目录：博文标题
│           ├── 视频1.mp4
│           ├── 视频2.mp4
│           └── ...
│
├── video-1/                        # 处理后的视频目录（HLS格式）
│   └── 视频分组/
│       └── 博文标题/
│           ├── 视频1/
│           │   ├── playlist.m3u8   # HLS播放列表
│           │   ├── segment_000.ts  # 视频片段
│           │   ├── segment_001.ts
│           │   └── ...
│           ├── 视频2/
│           │   └── ...
│           ├── 视频1_cover.jpg     # 视频封面（第1秒画面）
│           └── metadata.json       # 元数据文件
│
└── astro-melody-starter/
    └── src/
        └── content/
            └── videos/             # 视频博文Markdown文件
                └── 视频分组/
                    └── 博文标题.md

R2存储结构：
https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/video/
└── 视频分组/
    └── 博文标题/
        ├── 视频1/
        │   ├── playlist.m3u8
        │   ├── segment_000.ts
        │   └── ...
        ├── 视频2/
        │   └── ...
        ├── 视频1_cover.jpg
        └── metadata.json
```

### 步骤详解

#### 步骤1：准备视频

将视频放入 `video/` 目录，按分组和博文标题组织：

```
video/
└── 紧急企划/                       # 视频分组
    ├── 测试视频/                   # 博文标题
    │   ├── video1.mp4
    │   └── video2.mp4
    └── 测试视频2/
        ├── video1.mp4
        └── video2.mp4
```

支持的视频格式：`.mp4`, `.mkv`, `.avi`, `.mov`, `.flv`, `.wmv`

#### 步骤2：转换为 HLS 格式

```powershell
cd C:\huang
python video_processor.py
```

脚本功能：
- 遍历 `video/` 目录下的所有分组和博文
- 使用 FFmpeg 将视频转换为 HLS 格式
  - 视频编码：H.264
  - 视频码率：1000kbps
  - 音频编码：AAC
  - 音频码率：根据原始音频码率自动调整（最高320kbps，最低128kbps，默认192kbps）
  - 分辨率限制：1080P（保持原始比例）
  - 每个视频切片时长：10秒
  - 输出 `.m3u8` 播放列表和 `.ts` 视频片段
- 自动提取视频封面（第1秒的画面）保存为 `.jpg` 文件
- 生成 `metadata.json` 元数据文件
- 输出到 `video-1/` 目录
- 记录转换历史到 `video_conversion_record.json`

生成的 metadata.json 格式：
```json
{
  "group_name": "紧急企划",
  "post_title": "测试视频",
  "video_count": 2,
  "cover": "视频1_cover.jpg",
  "videos": [
    {
      "name": "视频1",
      "original_file": "video1.mp4",
      "size": 12345678,
      "duration": 120.5,
      "width": 1920,
      "height": 1080,
      "codec": "h264",
      "audio_bit_rate": 192000,
      "hls_playlist": "视频1/playlist.m3u8"
    },
    {
      "name": "视频2",
      "original_file": "video2.mp4",
      "hls_playlist": "视频2/playlist.m3u8"
    }
  ]
}
```

#### 步骤3：上传到 R2 存储

```powershell
cd C:\huang
python upload_video_to_r2.py
```

脚本功能：
- 遍历 `video-1/` 目录下的所有分组和博文
- 使用 rclone 上传到 Cloudflare R2 存储
- 远程路径：`small/video/视频分组/博文标题/`
- R2 公开访问地址：`https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/video/`
- 防止重复上传：通过 `video_upload_record.json` 记录已上传的博文

#### 步骤4：生成博文 Markdown 文件

```powershell
cd C:\huang
python generate_video_posts.py
```

脚本功能：
- 遍历 `video-1/` 目录下的所有分组和博文
- 读取 `metadata.json` 获取视频信息
- 生成 `.md` 文件到 `src/content/videos/分组/`
- 视频路径使用 R2 公开地址
- 记录生成历史到 `video_post_generation_record.json`

生成的 Markdown 文件格式：
```markdown
---
title: 测试视频
series: 紧急企划
description: 测试视频 - 5个视频
pubDate: "2026-02-28"
videoCount: 5
videos:
  - name: "视频1"
    hlsUrl: https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/video/紧急企划/测试视频/视频1/playlist.m3u8
  - name: "视频2"
    hlsUrl: https://pub-58906530c3a643c1b5d1101b21b03114.r2.dev/video/紧急企划/测试视频/视频2/playlist.m3u8
---
```

#### 步骤5：构建并部署到 main 分支生产环境

```powershell

npx wrangler pages deploy dist --branch=main
```

打开浏览器访问：

## 四、注意事项

1. **依赖要求**：
   - Python 3.x
   - Pillow 库（图片处理）
   - FFmpeg（视频处理，需添加到系统PATH）
   - rclone（R2上传，需配置远程存储）

2. **跳过机制**：
   - 所有脚本都会跳过已处理的内容
   - 如需重新处理，删除对应的记录文件：
     - `conversion_record.json`（图片转换）
     - `video_conversion_record.json`（视频转换）
     - `video_upload_record.json`（视频上传）
     - `video_post_generation_record.json`（视频博文生成）

3. **热更新**：
   - 生成新博文后无需重启服务器
   - Astro Content Collections 会自动检测文件变化
