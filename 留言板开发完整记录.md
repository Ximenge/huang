# Astro + Cloudflare Pages 留言板开发完整记录

## 项目概述

在 Astro 静态网站（已部署在 Cloudflare Pages）上添加留言板功能，使用 Cloudflare KV 作为数据存储。

---

## 需求分析

### 功能需求
1. 在首页导航栏添加"留言板"入口（位于"视频"和"标签"之间）
2. 留言板页面包含：
   - 留言列表（显示内容、时间倒计时）
   - 留言表单（昵称、内容）
   - 时间显示格式："刚刚"、"X分钟前"、"X小时前"、"X天前"
3. 防刷屏机制
4. 使用 Cloudflare 资源，尽量简单易操作

### 技术选型
- **前端框架**: Astro
- **部署平台**: Cloudflare Pages（已部署）
- **数据存储**: Cloudflare KV
- **API**: Cloudflare Pages Functions（集成在 Pages 中）

  const now = Date.now();
  const diff = now - timestamp;
  
  const seconds = Math.floor(diff / 1000);
  const minutes = Math.floor(seconds / 60);
  const hours = Math.floor(minutes / 60);
  const days = Math.floor(hours / 24);

  if (minutes < 60) return `${minutes}分钟前`;

## 问题与解决方案

### 问题 1：KV not available 错误

#### 现象
部署后访问 `/api/comments/` 返回：
```json
{ "error": "KV not available" }
```

#### 第一次尝试：检查 wrangler.toml
**操作**：确认 `wrangler.toml` 中已添加 KV 绑定配置

**结果**：问题未解决

#### 第二次尝试：生产环境部署
**操作**：部署到生产环境分支（main）

```bash
npx wrangler pages deploy dist --branch=main
```

**结果**：生产环境可以正常工作，但预览环境仍然报错

**原因分析**：
- Cloudflare Pages 的预览环境（分支部署）不会自动继承生产环境的 KV 绑定
- 需要在 Dashboard 中手动为预览环境配置 KV 绑定

#### 第三次尝试：Dashboard 配置
**操作**：在 Cloudflare Dashboard 中：
1. 进入 Pages 项目 → Settings → Functions
2. 选择环境："预览"
3. 在 KV namespace bindings 中添加：
   - 变量名称：`GUESTBOOK`
   - KV 命名空间：`GUESTBOOK`

**结果**：仍然报错

#### 第四次尝试：修改代码获取 KV 的方式
**操作**：修改 `src/pages/api/comments.ts`，使用多种方式尝试获取 KV

**初始代码问题**：
```typescript
const GUESTBOOK = (request as any).env?.GUESTBOOK;  // 这种方式在 Pages Functions 中不正确
```

**修改后的代码**：
```typescript
// 获取 KV 的辅助函数
function getKV(context: any): KVNamespace | null {
  // 1. 从 context.locals.runtime.env 获取 (Cloudflare Pages)
  const runtime = context.locals?.runtime;
  if (runtime?.env?.GUESTBOOK) {
    return runtime.env.GUESTBOOK;
  }
  
  // 2. 从 context.env 获取
  if (context.env?.GUESTBOOK) {
    return context.env.GUESTBOOK;
  }
  
  // 3. 从 globalThis 获取
  const globalEnv = (globalThis as any).env;
  if (globalEnv?.GUESTBOOK) {
    return globalEnv.GUESTBOOK;
  }
  
  return null;
}
```

**结果**：TypeScript 编译错误

#### 第五次尝试：安装类型定义
**操作**：
1. 安装 Cloudflare Workers 类型定义

```bash
npm install -D @cloudflare/workers-types
```

2. 更新 `tsconfig.json`

```json
{
  "compilerOptions": {
    "types": ["@cloudflare/workers-types"]
  }
}
```

3. 修复类型错误（添加接口定义）

```typescript
interface CommentBody {
  nickname: string;
  content: string;
}

// 使用时
const body = await request.json() as CommentBody;
```

**结果**：构建成功，部署后正常工作！

### 问题 2：TypeScript 类型错误

#### 现象
构建时出现多个类型错误：
- `Property 'runtime' does not exist on type 'Locals'`
- `Cannot find name 'KVNamespace'`
- `'body' is of type 'unknown'`

#### 解决方案
1. **安装类型包**：`@cloudflare/workers-types`
2. **配置 tsconfig.json**：添加 `"types": ["@cloudflare/workers-types"]`
3. **添加类型断言**：
   - `const body = await request.json() as CommentBody;`
   - `const data = await response.json() as { error?: string; success?: boolean };`

## 最终代码结构

### API 接口（src/pages/api/comments.ts）

```typescript
import type { APIRoute } from 'astro';

export const prerender = false;

interface Comment {
  id: string;
  nickname: string;
  content: string;
  ip: string;
  timestamp: number;
  status: string;
}

interface CommentBody {
  nickname: string;
  content: string;
}

// 获取 KV 的辅助函数
function getKV(context: any): KVNamespace | null {
  const runtime = context.locals?.runtime;
  if (runtime?.env?.GUESTBOOK) {
    return runtime.env.GUESTBOOK;
  }
  
  if (context.env?.GUESTBOOK) {
    return context.env.GUESTBOOK;
  }
  
  const globalEnv = (globalThis as any).env;
  if (globalEnv?.GUESTBOOK) {
    return globalEnv.GUESTBOOK;
  }
  
  return null;
}

export const GET: APIRoute = async (context) => {
  try {
    const GUESTBOOK = getKV(context);
    
    if (!GUESTBOOK) {
      return new Response(JSON.stringify({ 
        error: 'KV not available', 
        debug: 'GUESTBOOK binding not found'
      }), {
        status: 500,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    const comments = await GUESTBOOK.get('comments:all', 'json') as Comment[] || [];
    return new Response(JSON.stringify(comments), {
      headers: { 'Content-Type': 'application/json' }
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: 'Failed to fetch comments' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' }
    });
  }
};

export const POST: APIRoute = async (context) => {
  try {
    const GUESTBOOK = getKV(context);
    
    if (!GUESTBOOK) {
      return new Response(JSON.stringify({ error: 'KV not available' }), {
        status: 500,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    const request = context.request;
    const body = await request.json() as CommentBody;
    const ip = request.headers.get('CF-Connecting-IP') || 'unknown';

    // 验证
    if (!body.nickname || !body.content) {
      return new Response(JSON.stringify({ error: '请填写昵称和内容' }), {
        status: 400,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    if (body.content.length < 2) {
      return new Response(JSON.stringify({ error: '内容太短了' }), {
        status: 400,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    if (body.content.length > 500) {
      return new Response(JSON.stringify({ error: '内容太长了，最多500字' }), {
        status: 400,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    // 防刷检查
    const lastSubmit = await GUESTBOOK.get(`rate_limit:${ip}`);
    if (lastSubmit && Date.now() - parseInt(lastSubmit) < 60000) {
      return new Response(JSON.stringify({ error: '提交太快了，请稍后再试' }), {
        status: 429,
        headers: { 'Content-Type': 'application/json' }
      });
    }

    // 获取现有留言
    const comments = await GUESTBOOK.get('comments:all', 'json') as Comment[] || [];
    
    // 添加新留言
    const newComment: Comment = {
      id: Date.now().toString(),
      nickname: body.nickname.substring(0, 20),
      content: body.content.substring(0, 500),
      ip,
      timestamp: Date.now(),
      status: 'approved'
    };

    comments.unshift(newComment);
    
    // 最多保留 100 条
    if (comments.length > 100) {
      comments.length = 100;
    }

    // 保存
    await GUESTBOOK.put('comments:all', JSON.stringify(comments));
    await GUESTBOOK.put(`rate_limit:${ip}`, Date.now().toString(), {
      expirationTtl: 60
    });

    return new Response(JSON.stringify({ success: true, comment: newComment }), {
      headers: { 'Content-Type': 'application/json' }
    });
  } catch (error) {
    return new Response(JSON.stringify({ error: 'Failed to submit comment' }), {
      status: 500,
      headers: { 'Content-Type': 'application/json' }
    });
  }
};
```

## 关键知识点总结

### 1. Cloudflare Pages Functions 获取环境变量的正确方式

**错误方式**：
```typescript
const GUESTBOOK = (request as any).env?.GUESTBOOK;
```

**正确方式**：
```typescript
const runtime = context.locals?.runtime;
const GUESTBOOK = runtime?.env?.GUESTBOOK;
```

**原理**：
- Cloudflare Pages Functions 使用 `context.locals.runtime` 来访问运行时环境
- `runtime.env` 包含所有绑定的资源（KV、D1、R2 等）
- 这与传统 Worker 的 `request.env` 不同

### 2. 预览环境与生产环境的区别

**关键区别**：
- 生产环境自动使用 `wrangler.toml` 中的配置
- 预览环境需要在 Dashboard 中手动配置绑定
- 或者使用 `wrangler.toml` 中的 `[env.preview]` 配置

**配置方法**：
```toml
# 生产环境
[[kv_namespaces]]
binding = "GUESTBOOK"
id = "xxx"

# 预览环境
[env.preview]
[[env.preview.kv_namespaces]]
binding = "GUESTBOOK"
id = "xxx"
```

### 3. TypeScript 类型定义

**必须安装**：
```bash
npm install -D @cloudflare/workers-types
```

**配置 tsconfig.json**：
```json
{
  "compilerOptions": {
    "types": ["@cloudflare/workers-types"]
  }
}
```

**提供类型**：
- `KVNamespace` - KV 命名空间类型
- `Request` / `Response` - 扩展的请求响应类型
- 其他 Cloudflare 特定类型

## 总结

本次开发遇到的主要问题是 **KV 绑定在代码中无法正确获取**，根本原因是：

1. **对 Cloudflare Pages Functions 的运行时环境理解有误**
   - 错误地使用了 `request.env` 而不是 `context.locals.runtime.env`

2. **缺少必要的类型定义**
   - 没有安装 `@cloudflare/workers-types`
   - 导致 TypeScript 无法识别 `KVNamespace` 类型

3. **预览环境和生产环境的配置差异**
   - 预览环境需要额外配置或手动在 Dashboard 中设置

**最终解决方案**：
1. 安装 `@cloudflare/workers-types` 类型包
2. 更新 `tsconfig.json` 添加类型支持
3. 修改代码使用 `context.locals.runtime.env` 获取 KV
4. 为请求体和响应体添加类型断言

**核心原理**：
Cloudflare Pages Functions 是运行在 Cloudflare Workers 上的，但访问环境变量的方式与传统 Worker 略有不同。在 Pages Functions 中，需要通过 `context.locals.runtime` 来访问运行时环境，而不是直接通过 `request` 或全局变量。
